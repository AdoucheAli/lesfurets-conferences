Jenkins 2.0 and pipelines

What is it?

Describes all the job process by code

Master and agents

Jenkins DSL based on groovy:

  • stage, node, parallel
  • bat/sh, input, load, withCredentials
  • possibility to call any Jenkins plugin

node('linux') {
    stage('Packaging') {
        checkout scm
        sh 'mvn clean install'
    }
    stage('Deploy') {
        sh 'mvn jetty:run'
    }
 }
 stage ('Post deploy') {
    parallel(
        'Slack': {
            node() { notifySlack('Deployment done') }
        },
        'Update Jira': {
            node() { sh 'update_jira_tickets.sh' }
        }
 }

How to declare a job

Pipeline: a groovy file

Multi-branch pipeline: Jenkinsfile in you repository root

Pipelines? Why?

  • Use the power of Groovy
  • Version your pipeline code with Git
  • No damage caused by accidental click
  • Steps parallelization
  • Let the developers maintain their jobs
  • Better UX

JenkinsPipelineUnit

Pipeline code is versioned

But we want to check what will be executed as well

Example: the Job


def execute() {
    node() {
        def utils = load 'src/main/jenkins/lib/utils.jenkins'
        String revision = stage('Checkout') {
            checkout scm
            return utils.currentRevision()
        }
        gitlabBuilds(builds: ['build', 'test']) {
            stage('build') {
                gitlabCommitStatus('build') {
                    sh "mvn clean package -DskipTests -DgitRevision=$revision"
                }
            }

            stage('test') {
                gitlabCommitStatus('test') {
                    sh "mvn verify -DgitRevision=$revision"
                }
            }
        }
    }
}
return this
